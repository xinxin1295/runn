# 润 · 产品设计说明

给大厂痛风PM/程序员的工作陪伴者。

作者：信信 · AI实战课第一节作业

## 一 定位推导（用福格行为模型做）

课上说：`行为 B = 动机 M × 能力 A × 触发 T`，三者相乘，任何一项为0都不发生。

对我锁定的这类人来说：

| 要素 | 现状 | 解释 |
|---|---|---|
| 动机 M | 高 | 已经痛过一次，怕再痛，也在吃药 |
| 能力 A | 高 | 工位就有水杯，茶水间不远 |
| 触发 T | 极低 | 会议连轴、注意力锁死在屏幕、Slack响不停，根本想不起来 |

跟师兄课上那版一样，缺的是触发。但**触发方案不能照抄**，因为师兄面向的是自由职业者，我面向的是大厂人。

## 二 对师兄那版的差异化

师兄原方案：桌面轻提醒+固定间隔弹窗，解决"闹钟麻烦+吵"两个缺陷。

大厂PM/程序员的现实：即使是轻提醒，只要在会议/深度专注时弹出，就是打扰。他们的反应是**关掉整个应用**。所以真正要解的不只是「闹钟」的缺陷，还有一个师兄没提的问题——**「打扰时机」的错配**。

对症下药：

| 师兄那版 | 润这版 |
|---|---|
| 固定间隔提醒 | 按工作状态调节（会议中完全静默） |
| 记录喝了多少 | 记录"关节健康度反向减压"，让不喝的代价可见 |
| 通用喝水提醒 | 痛风专项：目标2500ml（比通用1500ml高）+ 急性发作日历 |
| 弹窗提醒 | 屏幕呼吸感提示，边缘光晕，不劫持注意力 |
| 关掉即关掉 | 关闭按钮设置情感成本挽留（不道德绑架） |

## 三 目标用户画像

**李工**，34岁，某大厂PM，坐标北京望京。产品双月OKR紧、日均会议6场、周三是需求评审日一天8场。

痛风病史3年，最近一次发作是三个月前赶大版本熬夜后凌晨2点脚趾剧痛，请了3天病假、错过了一个关键评审。他知道要多喝水但每天下班回想才发现只喝了一杯咖啡+两口水。装过大姨妈提醒式的喝水APP，第三天删了，理由是"开会时突然叮咚，尴尬"。

李工的关键诉求：
- 不能社死（会议中不能响）
- 不想被当孩子哄（不接受"该喝水了！"这种命令）
- 需要看到不喝的代价（不然大脑觉得"下次再喝也一样"）
- 愿意为身体投入，但不愿意为"仪式感"浪费时间

## 四 产品哲学（三条底线）

1. **不做警察做家人**：所有话术观察式+邀请式，禁止命令式与惊叹号
2. **让不喝的代价可见**：不只显示喝了多少，还显示按此频率下一次痛风大概什么时候
3. **静默感知>主动打扰**：优先靠视觉环境（呼吸光、水波）传达存在感，通知只在他能接的时刻响

## 五 功能设计详解

### 5.1 主视图 · 水面呼吸感

页面像一片静水。中央一个大水盂SVG，里面的水位随今日饮水量实时上涨。水面有慢波纹动画（周期6秒）。整个页面配色晨雾感，视觉噪音极低。

上方一行日期与农历（"庚寅日 · 甲午月"，参考中医"辰时喝水最利关节"的传统，只显示不强推）。

### 5.2 打卡三档

一排三个按钮：`一小口 125ml` / `一整杯 250ml` / `大杯 500ml`。点击后水位上涨动画，底部弹出一句从鼓励语库随机抽的话（后面第六节详列）。

不做精准记录，参考课上要点：喝水的主次矛盾是触发，不是精确容量。

### 5.3 状态切换 · 三态

顶部三个可切态：`会议中 / 深度专注 / 空隙时刻`。默认根据时间推测：9-10点/13-14点/17-18点=空隙，其余=深度专注。用户可以覆盖。

- 会议中：一切通知静默，页面呼吸放缓，边缘光晕熄灭
- 深度专注：只在页面边缘出现极淡光晕（1像素高、20%透明度、缓慢明暗），不弹通知
- 空隙时刻：允许桌面通知+页面主动提示

### 5.4 关节健康仪表

页面下方一个"关节镇"面板，画出大脚趾、膝、肘三处关节剪影。三个数字：
- 今日饮水累计
- 连续达标天数（水滴计数）
- 距下次可能急性发作 X 天（简单估算：连续达标天数越多，这个数字越大；断卡则减少）

这个"距下次" 数字是核心留人机制。看到 42 天比看到"今日已饮 500ml"更能打动痛风患者。

### 5.5 记录回顾

底部横向7天柱状图，每天一根竖水柱，达标日多一层青色，未达标日灰白。悬停显示当日ml数。

### 5.6 通知机制

- 默认45分钟触发一次（可在设置里改30/45/60/90）
- 只在"空隙时刻"或"深度专注"状态触发
- 会议中静默期结束30秒后补一次
- 通知内容不是"喝水吧"，是观察式话术（见6.1）

### 5.7 关闭挽留（关键）

用户点右上角关闭按钮时，弹出对话框：

> 你今天已经陪自己饮了 X 杯。
> 如果按这个节奏坚持，下一次急性发作可能延后到 42 天后。
> 现在真的要走吗？
> `我今天要请假` / `再留一天`

按 `再留一天` 回到主视图。按 `我今天要请假` 关闭页面，明天再打开时问一句"回来了？欢迎"。

这不是道德绑架，是让离开的动作有意义。让 close 从下意识变成有意识。

### 5.8 睡眠模式

0点到6点自动进入睡眠模式，通知禁用，页面变最深底色，中央出现一句"你也该歇了"。

### 5.9 开机自启

在设置里内置三步引导：
1. 按 `Win+R` 输入 `shell:startup` 回车
2. 打开的文件夹放一个 `润.html` 的快捷方式
3. 下次开机浏览器自动打开这个页面

这样每天开机就打开，无需用户主动记得。

## 六 内容库

### 6.1 通知话术（观察+邀请式，禁止命令式）

从下列中随机抽取，会根据一天内已触发次数微调语气（越晚越温柔）：

- 你的键盘敲了 47 分钟，允许自己端起杯子吗
- 会议刚散，是不是可以给自己 30 秒
- 距上次饮水 52 分钟，尿酸浓度正在缓慢上升
- 一小口水，就是给关节一次减压
- 手边有杯子就行，冷的也可以
- 昨晚睡得晚，今天更该多喝一点
- 空调开着，喝水能顶回来一半
- 深呼吸一下，端起杯子
（≥30条，产品HTML里给出完整数组）

### 6.2 打卡后的鼓励语

- 这 125ml，会去到你手指的每一个关节
- 好，多留一层护垫在膝盖里
- 一杯水的时间够写完一句 PRD
- 今天第 X 杯，血尿酸浓度往下走一点点
- 昨天连续达标，今天继续
- 会议前记得再来一口

### 6.3 关闭挽留话术

见 5.7。

## 七 视觉规范

### 7.1 配色

```css
--ru-tianqing: #a8c4c9;   /* 汝窑天青，主色 */
--ru-yuebai: #f4f1ea;     /* 月白，主背景 */
--ru-xuanzhi: #efe9dc;    /* 宣纸米，卡片背景 */
--ru-songyan: #2c3336;    /* 松烟墨，正文 */
--ru-qiuxiang: #c9a961;   /* 秋香黄，柔警告 */
--ru-feihong: #a3494a;    /* 绯红，急性发作提示，节制使用 */
--ru-shuiying: rgba(168,196,201,0.35); /* 水影，半透明覆盖 */
```

### 7.2 字体

- 中文：Noto Serif SC（思源宋体），字重400；标题字重500不加粗
- 英文/数字：Fraunces（衬线），字重300
- 通过Google Fonts加载（作业级用CDN即可）

### 7.3 动效

- 水波纹SVG：6秒一个周期
- 边缘光晕：8秒一个周期，透明度10-25%之间往复
- 打卡按钮hover：0.3秒缓入
- 页面加载：中央水盂从底部漫上来，1.2秒
- 通知弹出：从顶部缓降，0.8秒
- 全部使用 `cubic-bezier(0.4, 0, 0.2, 1)` 缓动

### 7.4 布局

- 单页无滚动
- 桌面端最小1024x768，居中显示，两侧留大空白
- 中央水盂占视线中心
- 状态切换在顶部一行
- 打卡按钮在水盂正下方一横排
- 关节镇在打卡按钮下方，横向排列
- 7天回顾在页面底部
- 移动端做单列适配但作业级不重点做

## 八 技术实现

- 纯HTML+CSS+原生JS单文件，无框架
- 数据存 localStorage，字段：`ru.today.ml`、`ru.streak.days`、`ru.lastCheckin.ts`、`ru.state`、`ru.settings.interval` 等
- 定时用 setInterval，注意页面隐藏时用 Page Visibility API 处理
- 桌面通知用 Notification API，首次进入询问权限
- 保活：加 Wake Lock API 让屏幕不锁（用户可选）
- 水盂用SVG手绘，波纹用SVG的animate标签或CSS keyframes

## 十 用课上的定位来验收

课上第4铁律：**用定位去验收**。

定位是"给大厂痛风PM/程序员的痛风友好工作陪伴者"。验收清单：

- [ ] 会议中静音吗（差异化核心，不静音就是没做对）
- [ ] 提示话术有没有一句是命令式（有=没做对）
- [ ] 关闭按钮点下去是不是弹挽留（不弹=普通APP）
- [ ] 有没有显示"距下次急性发作 X 天"（没有=没抓住痛风人的心）
- [ ] 视觉上有没有紫色渐变或Inter字体（有=陷入AI爆款）
- [ ] 打开是不是能双击HTML直接跑（不能=作业不合格）
- [ ] 有没有教怎么开机自启（没有=不能持续使用）

以上全过才算符合定位。
